# DESIGN.md - 群编年史

## 1. Visual Theme & Atmosphere
- 整体气质继承 `tatalab.ai`：明亮、亲和、叙事感强、像一块块被精心摆放的故事展板。
- 不做冷硬科技风，不做纪念馆式沉重黑白，不做标准 SaaS 模板感。
- 页面应让人感到“这里被认真整理过，但仍然柔软”。
- 站点的核心情绪：可信、温柔、克制公开、缓慢生长。

## 2. Color Palette & Roles
- `slate-50`：主背景，给留白与呼吸感。
- `white`：卡片表面与高亮容器。
- `slate-900 / slate-800 / slate-600 / slate-500`：标题、正文、辅助文本层级。
- `blue-600`：主品牌色，承担按钮、链接、导航高亮、关键强调。
- `cyan-500 / cyan-100`：用于时间线与流动感较强的内容。
- `indigo-500 / indigo-100`：用于里程碑和结构性模块。
- `pink-500 / pink-100`：用于人物、陪伴、情绪性内容。
- `emerald-500 / emerald-100`：用于原则、公开边界、安全提示。

## 3. Typography Rules
- 采用现代无衬线中文阅读风格，标题要足够厚重，正文保持温和、通透。
- Hero 标题应有强对比和局部渐变强调，但不要浮夸。
- 正文行高保持宽松，优先 1.75 附近的阅读感。
- 小标签、导航和说明文字可以更轻更紧，突出层级。

## 4. Component Styling
- 卡片：`rounded-3xl` 优先，白底、浅边框、轻阴影。
- 按钮：主按钮蓝底白字，次按钮白底灰字带边框，hover 仅做轻微提亮。
- 标签：圆角胶囊样式，轻底色 + 深色文字。
- 时间线：使用竖向时间轴、彩色圆点、阶段卡片、精选片段引用块。
- 人物卡：顶部可有浅色角标或柔和色块，不做头像墙式密集排布。

## 5. Layout Principles
- 使用故事型长页布局：Hero -> 时间线 -> 里程碑 -> 人物 -> 群画像 -> 原则。
- 留白要充足，卡片间距统一，避免内容过密。
- 章节头部可配圆形模糊色团，建立轻微“记忆气泡 / 实验室”氛围。
- 首屏既要有情绪，也要快速讲清网站是什么、为什么存在。

## 6. Depth & Elevation
- 阴影轻而软，避免重阴影和强烈拟物。
- 背景层次更多通过浅色块、模糊色团、边框和留白来建立，而不是靠厚重投影。
- 导航使用半透明白底 + backdrop blur，表现轻盈的悬浮感。

## 7. Do's and Don'ts
- Do：看起来像“认真整理过的群体记忆”。
- Do：把温柔和可信放在视觉第一优先级。
- Do：让信息密度服务叙事，不为了炫技堆组件。
- Don't：不要变成深色科技官网。
- Don't：不要出现过度商业化 CTA、价格卡、营销冲动文案。
- Don't：不要把公开编年史做成完整聊天记录转储页。

## 8. Responsive Behavior
- 移动端保持单列阅读优先，时间线改为紧凑单列卡片。
- 导航在小屏只保留品牌与最核心入口，不强求完整横向菜单。
- Hero 卡片与人物卡在移动端应先保证阅读顺序，再考虑视觉层次。
- 所有触控元素保持舒适点击面积。

## 9. Agent Prompt Guide
- 如果让 AI 继续生成页面，请明确说：
  - “继承 tatalab.ai 式的浅色叙事风格，不要做成模板 SaaS 页面。”
  - “白底圆角大卡片、浅灰边框、蓝色主品牌、粉/青/靛/绿作为章节强调色。”
  - “整体像公开但克制的群体记忆展板，重点是温柔、可信、纪念感。”
